Hamster-Fi

Wi-Fi роутер на Raspberry Pi з режимами AP / Station / Bridge

1. Ідея проєкту

Hamster-Fi — це навчальний проєкт, який реалізує класичний Wi-Fi роутер на базі Raspberry Pi з простим інтерфейсом для користувача.

Основна ідея — зробити «Wi-Fi для хомʼячків»: користувач може налаштувати мережу без знання Linux, мереж, NAT чи firewall.

Фокус: простота, готові профілі, автоматизація

2. Підтримуване обладнання

  • Raspberry Pi (орієнтовано на Pi 4 / Pi 5)
  • Вбудований Wi-Fi адаптер
  • Ethernet порт
  • Raspberry Pi OS (Debian-based Linux)

3. Режими роботи

3.1 AP mode (Router)

  • Raspberry Pi створює власну Wi-Fi мережу
  • Клієнти отримують IP через DHCP
  • Використовується NAT
  • WAN може бути:
    • Ethernet (eth0)
    • Wi-Fi (wlan0)

3.2 Station mode

  • Raspberry Pi підключається до існуючої Wi-Fi мережі як клієнт
  • Wi-Fi використовується як WAN
  • Пристрій виступає маршрутизатором для LAN

3.3 Bridge mode

  • Wi-Fi і Ethernet обʼєднуються в один L2-міст
  • Без NAT, без маршрутизації
  • Пристрій поводиться як «розумний подовжувач мережі»

4. WAN конфігурація

  • Dynamic IP (DHCP)
  • Static IP:
    • IP адреса
    • Шлюз
    • DNS

5. WLAN конфігурація

  • SSID
  • Pre-shared key (WPA2/WPA3)
  • Канал
  • Країна (regulatory domain)

6. Мережева логіка

  • DHCP сервер: dnsmasq
  • NAT + Firewall: nftables
  • HostAP: hostapd
  • IP forwarding через sysctl
Firewall мінімалістичний:
  • Drop за замовчуванням
  • Allow established/related
  • Allow LAN → WAN
  • Allow DHCP, DNS

7. Quick Wizard

Quick Wizard — основний спосіб налаштування. Користувач відповідає лише на базові питання:

  • Режим роботи
  • Тип WAN
  • DHCP або Static
  • Назва Wi-Fi та пароль

8. Advanced Mode

  • Редагування повного YAML-конфігу
  • Можливість змінити будь-який параметр
  • Валідація конфігурації

9. Збереження і скидання

  • Save config — конфіг зберігається в /etc/hamster-fi/config.yaml
  • Reset config — повернення до дефолтних значень
  • Factory defaults — повне скидання

10. UI

  • Web UI (FastAPI)
  • CLI утиліти для діагностики
  • Recovery-скрипт

11. Тест-план

Use-case 1

Користувач хоче створити власну Wi-Fi мережу з Ethernet інтернетом

Use-case 2

Користувач використовує Wi-Fi як WAN

Use-case 3

Користувач хоче розширити існуючу мережу через Bridge

12. Структура проєкту

  • hamsterfi/ — основний застосунок
  • templates/ — UI
  • system/ — мережеві скрипти
  • scripts/ — встановлення та recovery
  • docs/ — документація
